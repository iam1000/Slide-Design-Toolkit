# 슬라이드 디자인 툴킷: 새로운 디자인 추가 가이드

이 프로젝트는 UI 렌더링 코드와 데이터 구조가 분리되도록 설계되어 있어, 별도의 복잡한 프로그래밍 지식 없이도 몇 가지 설정만 추가하면 새로운 업종과 프롬프트를 쉽게 확장할 수 있습니다.

새로운 메뉴(업종)와 프롬프트를 추가하려면 크게 **데이터 추가**와 **연결(매핑)** 2단계만 거치면 됩니다.

---

## 1. 왼쪽 세부 업종 메뉴 변경/추가하기

사이드바(왼쪽 메뉴)에 나타나는 "대분류"와 "세부 업종" 리스트는 데이터 파일에 배열 객체 형태로 저장되어 있습니다.

**📂 수정 대상 파일:** `src/data/prompts-data.js`

해당 파일의 최상단을 보시면 `export const categories = [...]` 구조가 있습니다.
새로운 업종을 추가하려면 해당 대분류(id)를 찾아 `industries` 배열 안에 텍스트를 추가해 주면 됩니다.

```javascript
export const categories = [
  {
    id: "it",
    name: "IT/스타트업",
    emoji: "💻",
    industries: ["소프트웨어", "앱개발", "AI", "게임", "신규업종"], // '신규업종' 추가!
  },
  // ... (다른 리스트들)
];
```

> **Tip:** 아예 새로운 대분류 카테고리를 추가하고 싶다면 `{ id: 'new_id', name: '새 대분류', emoji: '🌟', industries: ['테스트'] }` 형식을 `categories` 배열 마지막에 통째로 추가하면 됩니다.

---

## 2. 해당 업종의 7선 프롬프트 내용 작성하기

사이드바에서 사용자가 방금 만든 메뉴를 눌렀을 때 화면 중앙에 표시될 **7가지 스타일 프롬프트 데이터 배열**을 만들어야 합니다.

**📂 수정 대상 파일:** `src/data/prompts-data.js`

파일 스크롤을 밑으로 내려서 기존 프롬프트 변수들이 있는 공간(아무 곳이나 무방함)에 다음과 같은 형태로 `export const` 데이터를 새로 만들어줍니다. 배열의 이름은 직관적으로 알아볼 수 있게 대문자로 작성합니다. (예: `NEW_INDUSTRY_PROMPTS`)

```javascript
export const NEW_INDUSTRY_PROMPTS = [
  {
    name: "미니멀 디자인", // 카드 상단 타이틀
    type: "Clean & Simple Style", // 카드 서브 타이틀
    colors: ["#FFFFFF", "#000000", "#1E90FF"], // 하단 컬러칩 3개 색상코드
    cardBg: "linear-gradient(135deg, rgba(30,144,255,0.08), rgba(0,0,0,0.05))",
    numBg: "rgba(30,144,255,0.1)", // 상단 숫자 배경 (선택)
    numColor: "#1E90FF", // 상단 숫자 색상
    prompt: `[NotebookLM Slide Design Guide]
Role : 신규 업종 전문가
Type : 미니멀 슬라이드

이곳에 복사할 실제 프롬프트 내용을 자유롭게 작성하세요.`,
  },
  // 동일한 형태로 두 번째, 세 번째 데이터 추가... (총 7개)
];
```

## 3. 새로 만든 프롬프트 데이터를 UI 기능에 연결하기 (매핑)

사용자가 왼쪽 메뉴('신규업종')를 클릭했을 때, 2번에서 만든 `NEW_INDUSTRY_PROMPTS` 배열을 화면에 뿌려주도록 매칭해주는 작업입니다.

**📂 수정 대상 파일:** `src/App.jsx`

이 파일 안쪽 영역에 보면 `const generatePrompts = (catName, industryName) => {` 라는 함수가 있습니다.
이 내부의 `switch (industryName)` 블록 안쪽 내용에 아래와 같이 방금 추가한 메뉴 텍스트와 데이터 변수명을 추가해줍니다.

```javascript
const generatePrompts = (catName, industryName) => {
  switch (industryName) {
    case "반도체":
      return PromptsData.SEMICONDUCTOR_PROMPTS;
    case "자동차":
      return PromptsData.AUTOMOTIVE_PROMPTS;
    // ... (생략) ...

    /* 👇 여기에 새로운 케이스를 추가합니다 */
    case "신규업종":
      return PromptsData.NEW_INDUSTRY_PROMPTS;

    default:
      return PromptsData.DESIGNER_BEST_7 || [];
  }
};
```

## ✅ 요약 및 저장 확인

1. `prompts-data.js`의 `categories` 리스트에 노출할 메뉴 이름 추가. (화면 왼쪽 좌측 메뉴 생성됨)
2. `prompts-data.js` 내에 노출시킬 내용을 담은 `export const 대문자_변수명 = [...]` 데이터 생성.
3. `App.jsx`의 `generatePrompts` 의 switch 구문에 `case '새 메뉴명': return 가져올_변수명` 한 줄 추가.

코드를 저장하시기만 하면 Vite(개발 서버)가 감지해서 즉시 변경된 화면 및 작동을 보여줍니다. 이상이 없는지 로컬 호스트 화면에서 클릭하여 확인한 후, GitHub에 푸시(Push)하시면 끝납니다.
